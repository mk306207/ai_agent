CloudSync API - Authentication Guide
=====================================

Overview
--------
CloudSync API uses OAuth 2.0 Bearer token authentication for secure access.
All API requests must include a valid authentication token in the request header.

Authentication Methods
----------------------
1. API Keys (Recommended for server-to-server)
   - Generate keys from Dashboard > Settings > API Keys
   - Include in header: Authorization: Bearer YOUR_API_KEY
   - Keys expire after 12 months

2. OAuth 2.0 (For user applications)
   - Redirect users to: https://auth.cloudsync.io/oauth/authorize
   - Exchange authorization code for access token
   - Tokens expire after 1 hour, refresh tokens valid for 30 days

Generating API Keys
-------------------
Step 1: Log in to CloudSync Dashboard (https://dashboard.cloudsync.io)
Step 2: Navigate to Settings > API Keys section
Step 3: Click "Generate New Key" button
Step 4: Name your key (e.g., "Production Server Key")
Step 5: Copy the key immediately - it will only be shown once!
Step 6: Store securely in environment variables

Using Authentication
--------------------
Include your API key in every request header:

curl -H "Authorization: Bearer cs_live_abc123xyz789" \
     https://api.cloudsync.io/v1/files

Rate Limits by Authentication Type
-----------------------------------
- API Key (Free tier): 100 requests/hour
- API Key (Pro tier): 1,000 requests/hour
- API Key (Enterprise): 10,000 requests/hour
- OAuth tokens: Same as account tier

Security Best Practices
------------------------
✓ Never commit API keys to version control (use .env files)
✓ Rotate keys every 90 days for security
✓ Use separate keys for development and production
✓ Revoke compromised keys immediately from dashboard
✓ Monitor key usage in Dashboard > API Analytics

Key Format
----------
API keys follow this pattern: cs_{environment}_{32_random_characters}
- cs_test_... (Test mode keys)
- cs_live_... (Production keys)
